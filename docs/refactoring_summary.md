## 2. 구조적 개선 (Architectural Improvements)

단순 기능 구현 위주였던 기존 코드를 **Clean Architecture** 원칙에 따라 재설계했습니다.

### 주요 변경 사항

- **UseCase 패턴 도입**: 라우터에 혼재되어 있던 비즈니스 로직(HTTPS 강제, 파라미터 변환 등)을 도메인 유즈케이스로 격리했습니다.
- **Repository-Interface 분리**: 도메인 계층이 인프라(Cosmos DB, Blob 등)의 상세 구현에 의존하지 않도록 인터페이스(ABC)를 도입했습니다.
- **상태 관리 모델화**: 단순 `dict` 기반 통신을 명확한 도메인 모델(`models.py`)로 전환하여 데이터 안정성을 확보했습니다.

## 3. 리팩토링 결과

- **성능**: 이벤트 루프를 멈추게 하는 블로킹 코드를 제거하여 비동기 처리 효율을 극대화했습니다.
- **유지보수**: 횡단 관심사(보안, 가드)와 비즈니스 관심사(유즈케이스)를 분리하여 코드 수정 범위를 최소화했습니다.
- **가독성**: Lint 규칙 준수 및 표준 패턴 적용으로 개발자 간 코드 협업 효율을 높였습니다.
