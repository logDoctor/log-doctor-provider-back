# Core Layer AI Instructions

## Role

Application global configuration, security policies, and non-domain specific engines.

## Rules

- **Config**: Access via `app.core.config.settings`. No direct environment variable access elsewhere.
- **Security**: 전역 보안 정책(예: AD 토큰 검증)을 여기서 구현합니다.
- **Multi-Strategy Auth**: `AUTH_METHOD` 설정에 따라 Managed Identity 또는 Client Secret 방식을 동적으로 선택하여 지원합니다.
- **Portability First**: 인증 및 DB 공급자 의존성을 `app/core/auth_provider.py` 및 `app/core/security.py`와 같은 인터페이스/팩토리 계층으로 추상화하여 특정 클라우드 벤더에 종속되지 않는 코드 구조를 추구합니다.
- **Shortcut Imports**: 핵심 모듈(`app/core`, `app/infra` 등)은 `__init__.py`를 통해 자주 사용되는 객체를 노출하여 임포트 경로를 단순화합니다.
- **Managed Identity**: 기본적으로 선호되는 방식이며, 로컬/백업용으로 `AUTH_METHOD="secret"`을 지원합니다.
- **Config**: 하드코딩된 시크릿을 저장하지 마세요. 선택적 `CLIENT_SECRET`을 사용하세요.
