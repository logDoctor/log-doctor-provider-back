# Core Layer AI Instructions

## Role

Application global configuration, security policies, exceptions, and non-domain specific engines.

## Rules

- **Config**: Access via `app.core.config.settings`. No direct environment variable access elsewhere.
- **Exceptions (`app/core/exceptions.py`)**:
  - Define all custom application exceptions here (`UnauthorizedException`, `ConflictException`, `NotFoundException`, etc.).
  - Inherit from standard patterns and allow the global exception handler to convert them into appropriate HTTP responses.
- **Security & Auth**: `app/core/auth` 패키지를 통해 중앙 집중화된 보안 정책을 관리합니다.
  - **Layer 1: Dependencies (`dependencies.py`)**: Spring의 `@Configuration`과 유사한 부품 공장(Bean Factory)입니다. 모든 서비스 객체의 생명주기와 의존성 조립을 담당합니다.
  - **Layer 2: Guards (`guards/`)**: Spring Security Interceptor와 유사한 보안 가드입니다. FastAPI `Depends`를 통해 라우터의 입구를 보호하며, HTTP 요청에서 데이터를 추출하고 정책을 집행합니다.
  - **Layer 3: Services (`services/`)**: 순수 비즈니스 로직(Pure Logic)입니다. 신원 파싱, 권한 검증, 토큰 발급 등의 핵심 기능을 수행합니다.
    - _Note on OBO Token_: MSAL 캐싱(In-Memory) 로직에 의지하며, Refresh Token을 DB에 직접 영구 보관하지 않는 Stateless 구조를 유지합니다. 발급 실패 시 `try-except` 블록으로 외부 장애를 안전하게 차단합니다.
- **Managed Identity**: 기본적으로 선호되는 방식이며, 로컬/백업용으로 `AUTH_METHOD="secret"`을 지원합니다.
- **Shortcut Imports**: `app/core/auth/__init__.py`를 통해 상위 수준의 가드와 유틸리티를 노출합니다.
