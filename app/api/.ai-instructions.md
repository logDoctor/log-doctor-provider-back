# API Layer AI Instructions

## Role

Application entry point, versioning management, and request/response orchestration.

## Rules

- **Versioning**: All endpoints must be registered under a versioned router (e.g., `app/api/v1/router.py`).
- **Pattern**:
  - Domain-specific endpoints must use **Class-Based View (CBV)** pattern with `fastapi-restful` inside `app/domains`.
  - Utility or cross-cutting endpoints (e.g., `template.py` loading static resources) can be standard `@router.get` functions inside `app/api`.
- **Logic Boundary**: No pure core business logic allowed. However, presentation/formatting logic (e.g., File I/O for templates, transforming JSON responses, building URLs from Request objects) is permissible here.
- **AOP & Guards**: Extract token parsing, permission checks, and identity validation into `Depends()` dependencies. API layer should only pass validated domain objects (like `Identity` or `tid`) downward.
