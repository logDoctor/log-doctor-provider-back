# Domain Layer AI Instructions

## Role

Core business logic and resource-driven domain models.

## Rules

- **Structure**: Each domain folder must contain:
  - `models.py`: Domain entities (Dataclasses) with `from_dict` and `to_dict`.
  - `router.py`: CBV-based endpoints.
  - `schemas.py`: Pydantic models for API Request/Response.
  - `dependencies.py`: Dependency injection providers.
  - `repository.py`: ABC interface + Concrete implementation.
  - `usecases/`: Pure business logic (e.g., `RegisterTenantUseCase`).
- **AOP Mapping**: Concrete repositories must use `@cosmos_repository(map_to=Entity)` to automate entity mapping and error handling.
- **Domain Logic**:
  - Business rules must reside within Domain Entities (Subject-oriented) as much as possible.
  - Use Cases must interact only with Domain Entities and Data Transfer Objects (DTOs/Schemas), not raw database dictionaries nor FastAPI `Request` objects.
- **Idempotency & Exceptions**:
  - Handle idempotency gracefully (e.g., do not upsert if the record already exists and shouldn't be altered).
  - Throw custom core exceptions (`ConflictException`, `NotFoundException`, `UnauthorizedException`) from `app.core.exceptions` instead of raising framework-specific HTTPExceptions directly in UseCases.
